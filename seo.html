Hi! Happy to help :)

I think this topic is often conflated with other items, which makes it hard. In particular, at Google we don't do anything special with the newer HTML5 tags.You can make up tags of your own, and they'll be +/- valid HTML, which is somewhat how we treat these "new" tags (they've been around for a long time, but anyway). Usually the process for changes in our systems in search is roughly: "we recognize that the web is doing something in a particular way, which is also beneficial for us" -> "we'll analyze advantages from using them" -> if there are significant advantages, we start using them. We wouldn't just change our systems based on how something could be used in theory, there has to be a real advantage to the end result (search quality) in order for us to add any complexity to our algorithms. That's still not the case with HTML5 -- we can recognize content on a page the same regardless of HTML5 usage or not.

The things this gets conflated with are creating a semantic structure on your page, or using valid HTML. Both of these do help us to better understand the content on a page, but it's not a matter of ranking higher just because they do this. For example, I can't read Polish, but I can read the letters on a web page in Polish, I can pull out the words, so I could point to that page if someone were to ask me for those words. However, if I could also read Polish, or if that page were in English, then I could better understand for which queries I might point at that page. Similarly, we can process all forms of broken HTML (the web is broken all over), and we can rank those pages in various ways, but it's easier if we have a clearer context. This is even more relevant nowadays with structured data on pages, where we need to understand which concepts on a page the structured data applies to.

Anyway, I think that's why you see such a variety of answers on this topic. Google doesn't use HTML5, we don't require semantic or valid HTML, its existence isn't a ranking factor, but it can make it easier to recognize the content, and show it for the relevant search results. (The trick is that "relevant search results" might be fewer than otherwise, but they're the ones that are useful for your pages. If you get 1000 people coming to your page and they laugh at a joke you have on there then leave, is it as valuable as getting just 10 people, if those 10 also buy something from there?
